# SocialSeal Social Plan Builder

## Overview

A SocialSeal social plan is an evidence-backed production and measurement plan. It should say what to create, why that content is needed, which keywords/topics it supports, and how SocialSeal will measure whether visibility improves.

The plan is not a generic calendar. Every planned content slot should connect to a search intent, opportunity, content job, and tracking group.

## Inputs

- opportunity analysis
- competitor/content pattern matrix
- tracking group IDs and keyword/topic scope
- platform and market priorities
- production capacity and available creators/assets
- reporting cadence

## Workflow

1. **Choose pillars from evidence.** Use 3-5 pillars based on keyword gaps and viewer jobs, not generic brand themes.
2. **Assign content jobs.** Each pillar should teach, compare, plan, reassure, show practical detail, or create mood.
3. **Choose formats.** Select formats evidenced by SocialSeal results: walkthrough, itinerary, first-timer guide, comparison, day-in-life, FAQ, etc.
4. **Build the plan table.** For each slot: platform, market, keyword/topic, content lens, format, hook direction, owner, asset needs, and tracking group.
5. **Create the brief queue.** Mark which slots need creator briefs, video concepts, asset plans, or generation prompts.
6. **Map measurement.** Each pillar maps to one or more tracking groups and expected signals: coverage, SOV, owned/creator split, example quality.
7. **Set review cadence.** Define when exports refresh and when the plan is adjusted.

## Output

A plan that can be pasted into a sheet or project tracker:

- pillar
- audience/search job
- target keywords/topics
- platform/market
- format
- content lens
- owner
- status
- measurement/tracking group
- evidence reference

Use `templates/social-plan-template.md` if available.

## Do / Don't

Do:

- make every slot traceable to SocialSeal evidence
- balance utility/practical and aspirational content
- keep cadence realistic for available creators/assets
- include a measurement column from the start

Don't:

- produce “3 posts per week” without saying what they answer
- plan content that cannot be measured in current tracking groups
- overfit to one competitor example
- use the plan as a posting automation spec

## Troubleshooting

- If the plan feels generic, return to the opportunity matrix and add keyword/topic evidence to each row.
- If there is no tracking group for a planned pillar, add a tracking-group-design task before publishing the plan.
- If production capacity is unknown, create a low/medium/high capacity version rather than one unrealistic plan.

## Verification Checklist

- [ ] Each row maps to a keyword/topic and tracking group.
- [ ] Each row has a content job and format.
- [ ] Evidence references are included.
- [ ] Cadence matches capacity.
- [ ] Plan includes next measurement date.
